# 贡献指南

感谢关注 TechBao Data Platform。本项目当前处于初始化阶段，欢迎围绕文档、部署方案、组件集成验证和最佳实践提交贡献。

## 贡献范围

- 文档完善：产品说明、架构说明、部署说明、组件集成说明。
- 部署模板：Docker Compose、Kubernetes、Helm、运维脚本等。
- 集成验证：SSO、数据同步、元数据采集、API 网关等验证记录。
- 问题反馈：组件兼容性、部署问题、安全风险和使用建议。

## 基本原则

- 不在平台仓库中提交具体业务代码。
- 优先通过成熟开源组件和标准协议解决问题。
- 新增组件前需要说明选型理由、替代方案和维护成本。
- 涉及架构方向的变更应补充或更新 ADR。
- 文档默认使用中文。

## 提交流程

1. 创建议题，说明背景、目标和影响范围。
2. 小步提交，保持变更聚焦。
3. 更新相关文档和示例。
4. 在提交说明中注明验证方式。
5. 等待维护者 review 后合并。

## 文档规范

- 使用 Markdown 编写。
- 标题层级清晰，避免过度嵌套。
- 对尚未完成的能力明确标注“规划中”“待验证”或“待实现”。
- 不声称组件已经集成，除非已有可复现的部署和验证记录。
