# ADR 0003：使用 Keycloak 作为统一认证和 SSO

## 状态

已接受

## 背景

平台规划中包含 Directus、Superset、DataHub、APISIX 等多个组件。若每个组件独立维护用户体系，会增加账号管理、权限治理和安全审计成本。

## 决策

使用 Keycloak 作为统一身份提供方，优先通过 OIDC/SAML 为各组件提供单点登录能力。

## 影响

- 用户、角色和客户端配置集中在 Keycloak 管理。
- 各组件保留自身权限模型，通过角色映射逐步统一。
- APISIX 后续可与 Keycloak 协同完成 API 认证。
- 需要沉淀 Realm、Client、Role、Group 的配置管理方式。

## 备选方案

- 组件独立账号：简单但难以治理。
- 使用企业现有身份源：适合生产环境，但本项目仍需要保留 Keycloak 作为开源参考方案。
- 自研认证中心：不符合少开发和集成优先原则。
